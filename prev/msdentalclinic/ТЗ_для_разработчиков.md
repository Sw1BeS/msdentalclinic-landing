## 1. Контекст і мета

На сайті клініки MS Dental запущено рекламну кампанію Facebook Ads для послуги **імплантації зубів**. Для цієї кампанії розроблено окрему посадкову сторінку `/implant` — вона вже готова і знаходиться в архіві.

**Що потрібно від вас:**
1. Розгорнути нову посадкову сторінку на сервері під URL `/implant`
2. Налаштувати передачу заявок із нової сторінки в існуючу БД із маркуванням джерела
3. Підключити другого адміністратора до Telegram-бота для отримання заявок
4. Оновити CRM-таблицю `/crm` для наочного відображення джерела заявок

Всі зміни вже реалізовані в коді — вам необхідно **розгорнути готовий архів** та провести верифікацію.

## 3. Зміни в коді — детальний опис

### 3.1 `main.go` — Go-сервер

#### 3.1.1 Нова структура `CallMe` (рядок 23–29)
```go
// БУЛО:
type CallMe struct {
    ID   int
    Name string
    Tel  string
    Time int64
}

// СТАЛО:
type CallMe struct {
    ID     int
    Name   string
    Tel    string
    Time   int64
    Source string   // ← нове поле
}
```

#### 3.1.2 Нова схема таблиці `callme` (рядок 32–43)
```sql
-- СТАЛО (нове поле source):
CREATE TABLE IF NOT EXISTS callme (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,
    name    TEXT NOT NULL,
    tel     TEXT NOT NULL,
    time    INTEGER NOT NULL,
    source  TEXT NOT NULL DEFAULT 'main'   -- ← нове поле
);
```

#### 3.1.3 Функція автоміграції БД `migrateTable` (рядок 46–78) — **НОВА**

Ця функція автоматично додає колонку `source` до **існуючої** бази SQLite при першому запуску нового бінарника — без ручних SQL-запитів і без втрати даних:

```go
func migrateTable(db *sql.DB) error {
    // Перевіряє наявність колонки source через PRAGMA table_info
    // Якщо колонки немає — додає: ALTER TABLE callme ADD COLUMN source TEXT NOT NULL DEFAULT 'main'
    // Логує факт міграції
}
```

> ✅ Вам нічого не потрібно робити з БД вручну — міграція відбудеться автоматично.

#### 3.1.4 Роут `/implant` — **НОВИЙ** (рядок 207–211)
```go
router.GET("/implant", func(w http.ResponseWriter, r *http.Request, _ httprouter.Params) {
    tmpl := template.Must(template.ParseFiles("site/implant.html"))
    tmpl.Execute(w, nil)
})
```

Після деплою сторінка буде доступна за адресою: `https://msdentalclinic.in.ua/implant`

#### 3.1.5 Ендпоінт `/callme/` — оновлений (рядок 270–301)

Тепер приймає необов'язкове поле `source` у JSON-тілі запиту:

```go
// СТАЛО:
type FormData struct {
    Name   string `json:"name"`
    Tel    string `json:"tel"`
    Source string `json:"source"`   // ← нове поле (необов'язкове)
}

// Якщо source не передано — встановлюється 'main' (зворотна сумісність)
if formData.Source == "" {
    formData.Source = "main"
}
```

> ✅ Зворотна сумісність повністю збережена — форми на поточному сайті продовжать працювати без змін.

## 4. Інструкція з деплою

### Крок 1. Зупиніть поточний процес

```bash
# Якщо systemd:
sudo systemctl stop msdental

# Якщо screen/tmux — знайдіть і зупиніть процес:
pkill msdental
# або
kill $(lsof -t -i:8080)
```

### Крок 2. Зробіть резервну копію БД

```bash
cp /path/to/msdentalclinic/db.sqlite /path/to/backups/db_backup_$(date +%Y%m%d).sqlite
```

> ⚠️ Це обов'язковий крок. Без нього у разі помилки можна втратити дані.

### Крок 3. Розпакуйте архів

```bash
# Перейдіть у директорію проєкту на сервері
cd /path/to/

# Розпакуйте архів (замінить існуючі файли)
unzip -o msdentalclinic_delivery_20260403.zip

# Структура після розпакування:
# /path/to/msdentalclinic/main.go
# /path/to/msdentalclinic/site/implant.html
# /path/to/msdentalclinic/site/static/...
# /path/to/msdentalclinic/templates/crm/callme_table.html
```

> ⚠️ Переконайтесь що `db.sqlite` НЕ потрапив в архів (і не перезаписався). Перевірте: `ls -la /path/to/msdentalclinic/db.sqlite`

### Крок 4. Зберіть новий бінарник

```bash
cd /path/to/msdentalclinic

# CGO потрібен для go-sqlite3
CGO_ENABLED=1 go build -o msdental .

# Перевірте що збірка пройшла успішно
echo $?   # має вивести: 0
```

> Якщо go не встановлений globally — використайте повний шлях: `/usr/local/go/bin/go build -o msdental .`

### Крок 5. Запустіть новий процес

```bash
# Якщо systemd:
sudo systemctl start msdental

# Якщо запускаєте вручну (screen):
screen -S msdental
./msdental
# Ctrl+A, D — відчепитись від screen
```

### Крок 6. Перевірте логи при старті

```bash
# systemd:
sudo journalctl -u msdental -n 20

# або screen:
screen -r msdental
```

Очікуваний лог при першому запуску на існуючій БД:
```
Migration: added 'source' column to callme table
Server listening on :8080
```

На сторінці встановлено `<meta name="robots" content="noindex, nofollow">`. Це correct behavior для рекламної посадкової сторінки — трафік туди йде виключно через платну рекламу, не через органіку.

### WAF залишається без змін

Файли WAF (`site/other/waf/`) не змінювались. Конфігурація reverse proxy залишається ідентичною: зовнішні запити проходять через WAF на порту `:8090`, далі проксуються на Go-сервер `localhost:8080`.

Не ламається нічого. Ендпоінт `/callme/` приймає той самий JSON що й раніше, просто тепер додатково читає поле `source`. Форми, які не передають `source`, отримають значення `'main'` за замовчуванням.
